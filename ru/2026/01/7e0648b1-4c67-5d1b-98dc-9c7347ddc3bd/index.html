Ошибка "450 4.7.1 Helo command rejected: Host not found", как понять?

  • 2026/01/28 15:05:53 UTC

В логах Postfix появляется ошибка <...>Helo command rejected: Host not found<...>. Что это значит?

Полный текст ошибки:

1

May 26 09:15:02 mx postfix/smtpd[12607]: NOQUEUE: reject: RCPT from smtp1.example.net[191.5.61.83]: 450 4.7.1 <site.corporate.ad>: Helo command rejected: Host not found; from=<mail@example.com> to=<mail@example.org> proto=ESMTP helo=<site.corporate.ad>

Проблема на стороне отправителя письма. Для хоста site.corporate.ad отсутствует PTR-запись. Чтобы разрешить хосту site.corporate.ad присылать Вам письма, необходимо добавить его в исключения.

Открыть /etc/postfix/helo_access.pcre, добавить:

1

/^site\.corporate\.ad$/ OK

…или если необходимо разрешить почту со всего домена .corporate.ad:

1

/\.corporate\.ad$/ OK

Перезагрузить Postfix:

1

systemctl restart postfix.service

Поиск

Используется нечёткий поиск (приблизительное сопоставление строк). Это метод поиска строк, которые приблизительно соответствуют поисковому шаблону.

Результаты поиска сортируются по точности совпадения с поисковым шаблоном. Результаты поиска, наиболее точно соответствующие запросу, находятся сверху, приблизительные результаты - снизу.
